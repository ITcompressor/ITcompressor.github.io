Посмотреть все изменения файла локально. При указании имени файла изменения будут показаны только для него. 
git diff
Просмотреть историю коммита. Может также использоваться для файла с git log -p my_file. Введите q, чтобы выйти. 
git log
Просмотреть, кто, что и когда изменил в my_file. 
git blame my_file
Показать журнал изменений в заголовке локального репозитория. Отличный вариант для поиска утраченных данных. 
git reflog
Отмена проиндексированных и непроиндексированных изменений с момента последнего коммита. (Revert git add .) 
git reset --hard HEAD
Отмена изменений my_commit 
git checkout my_commit
Отмена последствий изменений в my_commit. revert выполняет новый коммит после отмены изменений. 
git revert my_commit
Удаление неотслеживаемых файлов в локальной рабочей директории.
-n — флаг для пробного запуска, ничего не удаляется.
-f — флаг для удаления файлов.
-d — флаг для удаления неотслеживаемых директорий. 
git clean -n
Добавляем поэтапные изменения в последний коммит. 
git commit --amend
Отправка локальных тэгов в удаленный репозиторий. Хороший вариант для присвоения версий изменениям. 
git push my_remote --tags
Полезная надстройка команд для git 
git config --global user.name "Username"
git config --global user.email example@mail.com
git config --global alias.co checkout
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.br branch
git config --global alias.hist 'log --pretty=format:"%h %ad | %s%d [%an]" --graph --date=short'
git config --global alias.type 'cat-file -t'
git config --global alias.dump 'cat-file -p'
Откатить уже отправленные изменения: 
git reset --hard origin/master
git push -f
Добавление репозитория для отправки изменений 
git remote add origin git@bitbucket.org:any_user/any_repo.git
Можно использовать такое представление урла чтобы не использовать креды (url pattern with creds) 
https://user:pass@domain/repo
Переименовать ветку 
git branch -m master2 develop # - rename local branch
git push origin :master2 # - delete old remote branch
git push --set-upstream origin develop # push renamed branch and change origin to new
Посчитать количество строчек кода в *.java: 
git ls-files | grep -P ".*(java)" | xargs wc -l
